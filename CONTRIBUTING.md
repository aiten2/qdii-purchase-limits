# 参与贡献

欢迎修复解析问题、补充测试、改进跨平台兼容性或完善文档。

## 开发流程

1. Fork 本仓库并创建独立分支。
2. 修改代码，同时补充对应的离线测试。
3. 运行发布门禁：

```bash
node scripts/check-release.js
node scripts/check-git-history.js
```

4. 提交 Pull Request，说明问题、改动、验证结果和可能影响的数据来源。

## 数据与隐私要求

- 测试夹具必须使用最小必要内容，不提交整页抓取结果或大规模第三方数据集。
- 不提交 token、cookie、webhook、账号、真实交易记录或本机绝对路径。
- 新数据来源必须可公开访问，并在代码或文档中写明来源、抓取范围和失败降级方式。
- 不得绕过登录、验证码、访问控制或网站明确设置的技术限制。
- 不得把查询结果包装成基金推荐、收益承诺或交易信号。

## 代码要求

- 保持 Node.js 18 或更高版本可运行。
- 优先使用 Node.js 内置模块，新增依赖时说明必要性和许可证。
- 网络失败、页面结构变化和数据缺失必须明确暴露，不能静默输出可买结论。
- 自动化和通知必须保持默认关闭，由用户主动启用。
